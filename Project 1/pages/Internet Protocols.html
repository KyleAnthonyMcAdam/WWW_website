Internet Protocols

Internet Protocols. What are they?

Internet Protocols are very useful parts of the network layer communications protocol. These protocols help us perform numerous functions that may seem trivial but without them even something as simple as sending an email would be impossible. Here aer some example down below.

Internet Protocols:
Protocol Name Description of protocol
Transmission Control Protocol (TCP) TCP divides messages into packets when it is sent and then reassemble the packets upon arrival their destination
Internet Protocol (TCP) connectionless protocol but doesnâ€™t require a connection with the host in order to send or retrieve data. UDP transfers data in the form of diagrams
Post Office Protocol (POP3) POP3 is a protocol used to receive emails.
Simple Mail Transport Protocol SMTP is a protocol that is designed to send mail.
Hyper Text Transfer Protocol(HTTP) HTTP is a protocol made to transfer hypertext between two and more systems. Html documents are shared.
File Transfer Protocol(FTP) FTP duplicates files from one host to another. FTP uses like control process and data transfer process at both ends. It creates two different connections; one is used to transfer data and the other to control information.
Trivial File Transfer Protocol(TFTP) TFTP, like FTP, also shifts files but without any authentication. It also does not separate control and data information like FTP. TFTP does not incorporate any security features, this means that it is not a recommended protocol
